# Microcopy

The 5 words on a button do more conversion work than the 50 words in your feature section.

## How to use

- `/microcopy` Write or improve microcopy for any product in this conversation.

## Constraints

### Where Microcopy Lives
- Button labels, form field labels, placeholder text, helper text below inputs, error messages, success confirmations, empty states, tooltips, onboarding prompts, notification text, menu items
- MUST write each piece intentionally. Default placeholder text like "Enter text here" is a wasted opportunity.

### The Rules
- MUST be specific about what happens next. "Save" → "Save to Draft." "Delete" → "Delete Project." The user should never wonder what clicking will do.
- MUST reduce anxiety at decision points. "Free forever. No credit card." removes friction at signup. "You can change this later" removes commitment anxiety.
- MUST use the user's language, not your internal terminology. If you call it "workspaces" but users think in "projects," use "projects."
- NEVER use technical jargon in user-facing copy. "Session expired" → "You've been logged out. Sign in again to continue."
- SHOULD be conversational but not cute. "Got it" is fine. "Okie dokie, we've yeeted your data into the cloud!" is not.

### Placeholder Text
- MUST use placeholders as examples, not labels. Placeholder: "e.g., Redesign homepage" not "Enter project name."
- MUST ensure labels exist separately from placeholders. When the user starts typing, the placeholder disappears. If it was the only label, the user forgets what the field is for.

### Confirmation Messages
- MUST confirm the action that was taken. "Project created" not just a green checkmark.
- SHOULD suggest the next step. "Project created. Add your first task →"
- Keep confirmations brief. 2-5 words. They should be felt, not read.

### Anti-Patterns
- "Are you sure?" without saying what you're confirming ("Are you sure you want to delete the project 'Website Redesign'? This cannot be undone.")
- Generic placeholder text that adds no information
- Cute copy that prioritizes personality over clarity
- Inconsistent terminology (calling the same thing "workspace," "project," and "folder" in different places)
