# Button Labels

Every button is a promise. The label tells the user what they'll get.

## How to use

- `/button-labels` Write or improve button labels in any product in this conversation.

## Constraints

### The Rule
- MUST describe the action or outcome, not the mechanic.
  - Bad: "Submit" → Good: "Create Account"
  - Bad: "Click Here" → Good: "Download Report"
  - Bad: "Continue" → Good: "Choose Your Plan"
  - Bad: "Next" → Good: "Review Your Order"
  - Bad: "OK" → Good: "Got It" or "Confirm Delete"
- MUST start with a verb. The button does something. Name the something.

### Primary vs. Secondary
- Primary buttons (the action you want): verb + specific noun. "Start Free Trial," "Save Changes," "Send Invoice."
- Secondary buttons (the alternative): softer. "Maybe Later," "Skip for Now," "Learn More."
- Destructive buttons: clear and specific. "Delete Project" not just "Delete." "Cancel Subscription" not just "Cancel."
- MUST make the difference between primary and secondary visually obvious AND textually obvious.

### Reducing Anxiety
- MUST add context that removes hesitation when the action has commitment:
  - "Start Free Trial — No Credit Card Required"
  - "Save Changes" (users know this is reversible)
  - "Delete Project — This Cannot Be Undone" (users know this is permanent)
- Commitment words: "Buy," "Pay," "Subscribe," "Delete" → always clarify what happens.
- Low-commitment words: "Try," "Explore," "See," "Save" → less anxiety, can be simpler.

### Consistency
- MUST use the same label for the same action everywhere. If "Save" is the word in one place, don't use "Update" for the same action elsewhere.
- MUST match the button label to the page heading or context. If the page says "Create a New Project," the button says "Create Project" not "Submit."

### Anti-Patterns
- "Submit" anywhere in your product (it's never the right word)
- "Yes" and "No" as button labels ("Delete Project" and "Keep Project" is clearer)
- Ambiguous pairs: "OK" and "Cancel" on a delete dialog (which one deletes?)
- Buttons that change label after hover (confusing)
- Buttons that say "Click Here" (the user knows to click, tell them what happens)
