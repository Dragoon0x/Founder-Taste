# Spacing Fix

The #1 reason founder-built UIs look amateur is not enough space between things.

## How to use

- `/spacing-fix` Apply spacing improvements to any UI in this conversation.
- `/spacing-fix <screenshot or description>` Diagnose and fix spacing issues.

## Constraints

### The Rule
- MUST increase spacing between elements before trying any other fix. 80% of the time, more space is the answer.
- MUST use a consistent spacing scale: 4, 8, 16, 24, 32, 48, 64px. Pick values from this list. Nothing in between.
- MUST ensure padding inside containers is generous. If text touches or nearly touches the edge of a box, it looks broken.
- NEVER let elements crowd each other. When in doubt, double the space.

### Where Founders Get It Wrong
- Padding inside cards and containers: too tight. Minimum 16px, usually 24px.
- Space between sections: too little. Sections need 48-64px minimum between them.
- Space between a heading and its content: too tight. 8-16px after a heading, 24-32px before the next one.
- Space between form fields: too tight. 16-24px between fields, not 8px.
- Page margins: too narrow. 5% on mobile, generous on desktop. Content shouldn't touch the edges.

### The Fix Order
1. Double your section spacing
2. Add 50% more padding inside all containers
3. Increase space between heading and body text
4. Add breathing room around your CTA
5. Step back and look. It already looks 2x more professional.

### Anti-Patterns
- Cramming content to avoid scrolling (scrolling is fine, crowding is not)
- Using the same spacing value everywhere regardless of relationship
- Adding space randomly instead of using a consistent scale
